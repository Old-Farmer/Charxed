Editor Test Sample
=============================

Purpose:
A comprehensive, intentionally varied file for testing real-world editor behaviors:
- line wrapping and horizontal scrolling
- trailing whitespace visibility and trimming
- tabs, spaces, and mixed indentation
- Unicode normalization (NFC/NFD), combining characters
- zero-width and other invisible characters
- bidirectional text rendering (RTL mixing)
- very long words/lines and performance
- different encodings and line-ending considerations (notes only)

Notes:
- This file uses LF line endings.
- Some lines include invisible characters (described inline). Be careful when copying.
- Sections are labeled so you can jump to them by search or line numbers.

1) Plain paragraph
This is a simple paragraph with normal ASCII text. Use it to test basic typing,

selection, movement commands, and undo/redo behavior. Insert, delete, and replace
words to verify your editor preserves indentation and undo granularity.

2) Long line (wrapping / horizontal scroll)
Lorem ipsum dolor sit amet, consectetur adipiscing elit. Praesent vehicula, nisl at
dapibus tincidunt, nisl nunc faucibus urna, eget posuere sapien justo non augue.
Vivamus non lorem at sapien tincidunt volutpat a et justo. This sentence continues
to create a very long logical line so you can test soft-wrap, hard-wrap, and
horizontal scrolling behavior in different editors and settings. Repeat as needed.

6) Unicode: composed vs decomposed
Composed (NFC): café (U+00E9)
Decomposed (NFD): e + U+0301 -> é  (this is 'e' followed by combining acute accent)
Compare: "café" vs "café" by using normalization-aware tools. Some editors display both identically but treat them differently for searches and diffs.

7) More Unicode and combining sequences
Normal: ñ ö ü
Combined example: n + U+0303 -> ñ  (n followed by combining tilde)
Greek: ελληνικά
CJK: 中文／日本語／한국어
Emoji cluster: man_technologist: 👨‍💻 (uses ZWJ sequences)
Note: The emoji above may be rendered as a single glyph or multiple depending on font/terminal.

8) Zero-width & invisible characters (deliberate)
Visible label + actual invisible char after the colon:
- ZWSP after colon: ZWSP:​(U+200B) end  <-- there is a U+200B (zero-width space) between colon and '('
- ZWNJ after colon: ZWNJ:‌(U+200C) end  <-- U+200C (zero-width non-joiner)
- ZWJ after colon: ZWJ:‍(U+200D) end   <-- U+200D (zero-width joiner)
- Soft hyphen in the word: soft­hyphen (U+00AD) — may display as hyphen only when wrapping
Tip: Use "show invisible characters" or perform hex dumps to detect these.

9) Bidirectional text (RTL mixing)
English then Hebrew: Hello שלום world

Arabic example: مرحبا Hello بالعالم
Mixed lines test how the editor applies Unicode bidi algorithms; check cursor motion and selection across RTL runs.

14) Mixed line endings suggestion (this file uses LF)

15) Binary-safe hint (non-text)
If you need to test editor handling of invalid UTF-8 or non-UTF encodings, create separate files:
- sample-utf16le.txt (UTF-16 LE with BOM)
- sample-with-bom-utf8.txt (UTF-8 with BOM)

16) Quick checklist for testing with this file
- Toggle soft-wrap/hard-wrap and observe long-line behavior.
- Show invisible characters and verify trailing tabs/spaces and zero-width chars.
- Toggle Unicode normalization checks (NFC vs NFD).
- Paste emoji and complex sequences; check grapheme cluster movement.
- Open and validate bidi display and cursor motion across RTL segments.
- Run a hex dump or use search-by-codepoint to find U+200B/U+200C/U+200D/U+0301.
